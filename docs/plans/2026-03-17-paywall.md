# Blog Paywall Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a lightweight paywall to White's Jekyll blog with Stripe payments (WeChat Pay/Alipay/Card), Cloudflare Worker for secure content delivery, and email-based access restoration.

**Architecture:** Jekyll renders free preview only — premium content is extracted at build time, rendered to HTML, and uploaded to Cloudflare KV. A Cloudflare Worker (`pay.haloowhite.com`) handles Stripe Checkout, JWT-based auth, content serving, and email verification via Resend. No user accounts — Stripe purchase email is the identity.

**Tech Stack:** Jekyll 4.3, Cloudflare Workers + KV, Stripe Checkout, Resend (email), GitHub Actions

## Task 2: Build Script — Extract Premium Content

**File:** `scripts/extract-premium.mjs`

Node.js script that:
1. Reads all `_posts/*.md` files
2. For posts with `premium: true` in front matter
3. Splits content at `<!-- paywall -->` marker
4. Renders premium markdown to HTML using `marked` library
5. Outputs JSON files: `_premium_output/{slug}.json` with `{ html, title, slug }`
6. Replaces original post file: keeps only free content (above marker)

This runs in GitHub Actions BEFORE Jekyll build, so only free content is in the deployed HTML.

## Task 3: Cloudflare Worker — API Backend

**Files:** `worker/src/index.js`, `worker/wrangler.toml`, `worker/package.json`

### Endpoints:

| Route | Method | Purpose |
|-------|--------|---------|
| `/api/checkout` | POST | Create Stripe Checkout session |
| `/api/callback` | GET | Stripe redirect → issue JWT → redirect to article |
| `/api/content/:slug` | GET | Serve premium content (requires JWT) |
| `/api/restore` | POST | Send verification code to email |
| `/api/verify` | POST | Verify code → check Stripe → issue JWT |
| `/api/webhook` | POST | Stripe webhook (subscription events) |

### Environment Bindings:
- `PREMIUM_CONTENT` — KV namespace (stores rendered premium HTML)
- `VERIFY_CODES` — KV namespace (email verification codes, 10-min TTL)
- `STRIPE_SECRET_KEY` — Stripe API secret key
- `STRIPE_WEBHOOK_SECRET` — Stripe webhook signing secret
- `RESEND_API_KEY` — Resend email API key
- `JWT_SECRET` — HMAC-SHA256 secret for JWT signing
- `BLOG_URL` — `https://haloowhite.com`

### JWT Structure:
```json
{
  "email": "user@example.com",
  "type": "article|subscription",
  "articles": ["arkose-funcaptcha-reverse-tutorial"],
  "iat": 1234567890,
  "exp": 1234567890  // 30 days for article, subscription end date for sub
}
```

### Auth Flow:
1. User clicks "Unlock" → JS calls `POST /api/checkout` with `{ slug, type }`
2. Worker creates Stripe Checkout session (payment or subscription)
3. Returns `{ url }` → JS redirects to Stripe
4. After payment, Stripe redirects to `/api/callback?session_id=xxx`
5. Worker verifies session, creates JWT, stores in URL fragment
6. Redirects to `{BLOG_URL}/{article-path}#token=JWT`
7. Client JS extracts token from hash, stores in `localStorage`
8. Client JS fetches `GET /api/content/{slug}` with `Authorization: Bearer JWT`
9. Premium HTML rendered into page

### Access Restoration Flow:
1. User clicks "Restore access" → enters email
2. JS calls `POST /api/restore` with `{ email }`
3. Worker generates 6-digit code, stores in KV (10-min TTL)
4. Worker sends code via Resend API
5. User enters code → JS calls `POST /api/verify` with `{ email, code }`
6. Worker verifies code, queries Stripe for email's purchases
7. Returns JWT with all purchased article slugs + subscription status
8. Client JS stores JWT in localStorage

## Task 6: Email Service (Resend)

Part of the Cloudflare Worker (Task 3). Uses Resend API:

```javascript
await fetch('https://api.resend.com/emails', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${env.RESEND_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    from: 'White\'s Blog <noreply@haloowhite.com>',
    to: email,
    subject: '验证码 - White\'s Blog 访问恢复',
    html: `<p>您的验证码是：<strong>${code}</strong></p><p>有效期 10 分钟。</p>`
  })
})
```

**Setup required:**
- Create Resend account, add `haloowhite.com` domain
- Add DNS records (SPF, DKIM) for email deliverability
- Get API key → add to Worker secrets
